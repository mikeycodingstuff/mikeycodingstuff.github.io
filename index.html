Portfolio

Collector's App

An app built in PHP that displays a collection of albums from an SQL database.

Information about each album is displayed, and users can add a new album to the collection, and delete any album from the collection.

User inputs are sanitised and validated, making use of a custom regular expression.

A try catch block is used to handle errors, with dynamic messages set via GET data.

Film Quiz App

A film quiz game built in JavaScript, consuming a pre-built API with film data.

Once the game starts the player has 30 seconds to attempt to guess as many quotes as they can.

A score counter displays the correct guesses with a visual indicator when the player reaches a multiple of 5.

A hint button shows the correct film's release year when clicked, and an information button opens an instructions modal.

>Github >Live

React Email Client

An email client built in React consuming a pre-built API to fit a supplied design.

Displays the user's inbox as well as sent emails.

Email list shows a preview of all emails. The user can select a preview to open the complete email.

Styled using Sass and Bootstrap, the app utilises modern JavaScript practices such as fetch to interact with the back-end API, and hooks to handle data rendering.

OOP PHP listing app

An object-oriented PHP project, styled with bootstrap, that fetches and displays a list of biscuits from a MySQL database.

We used the hydrator entity pattern to populate the app with data and a viewhelper to display it on the front end

We managed namespacing and autoloading using Composer and attempted to follow SOLID principles where possible.

Testing was performed with PHPUnit and custom exception handling was used to deal with errors.

Academy Application Portal

We undertook some new feature work and bug fixes on a legacy codebase, a student portal system.

We added functionality to add/remove trainers and to link trainers to courses.

Our work involved validation, sanitisation and unit testing within a slim MVC PHP framework with a MySQL database.

Routing was used for API functionality on the back-end, and link tables were used in MySQL for relational data.
